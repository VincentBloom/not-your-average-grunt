4. Read and explain the following article by Martin Fowler: http://martinfowler.com/articles/richardsonMaturityModel.html. 20 points.

Richardson Maturity Model is composed of 4 levels:

  • Level 3: Hypermedia controls.
  • Level 2: HTTP Verbs.
  • Level 1: Resources.
  • Level 0: The Swamp of POX.

As complicated as this system may seem, in reality it is a very approachable and obvious system to follow to unserstand REST (Representational State Transfer).

Level 0

The simple use of HTTP (Hypertext Transfer Protocol) for the transportation of data between a client and a server. It uses simple synchronous communication with a single server resource being interacted with.

Level 1

Resources, in this case, represent the actual division of content based on conceptual boundaries. Different 'objects' must be in their very own location as they are not part of the domain as other objects. Basically introduces what can be defined as entities with their own resource locator.

Level 2

Instead of carrying out communication through simple HTTP POST (it can be done), HTTP comes with functions as verbs that can make the use of services more convenient. Each verb adds semantic value to the communication, which can make transfers more identifiable and erros easier to pinpoint.

Level 3

Hypermedia controls are the holy grail of HTTP. They allow developers to pave the way for user interaction without the need for extensive knowledge of the system. They create a practical workflow for all the resources interacting in the system. Although they are not proper parts of REST, they are important, as they add another dimmension to state transfers.

5. Explain why cloud computing is important.10 points.

Cloud computing is the greatest extension known yet to one of computing's most important machines ever built - the Turing Machine. This machine revolutionized the way we think of computation. The idea that one machine had infinite storage (just by switching out tapes) was amazing. Cloud computing (cloud storage in this case) builds on the idea that you can have as much storage capability and that it is practically infinite.

Cloud computing offers services that not every person can afford to have, have the space to house, have the team to maintain, have the time to build or have the money to run. It gives people the oportunity to run different types of computational tasks without the need to worry of how many resources they need. It makes it very convenient for any in need of cloud computing to use.

Cloud computing offers scientits the resources to run all sorts of operations, models, benchmarks to do their research on, using up as many virtual machines as needed to solve the problems proposed.

Cloud computing providers are very flexible and services can be tailored and customized to one's needs.
